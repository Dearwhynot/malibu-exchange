# AGENT.md — Project Kit

## EN

This folder stores reusable building blocks for Malibu Exchange and future WordPress backoffice projects.

It is part of the Malibu theme root, not a separate repository outside the theme.

### Rules
1. Do not invent architecture when a proven module already exists.
2. Prefer extracting and cleaning existing solutions over rewriting from scratch.
3. Keep every module small, isolated, and reusable.
4. Do not mix multiple concerns in one module.
5. Keep prompts, docs, and extracted code aligned.
6. Use the module structure consistently:
   - source
   - extracted
   - docs
   - archives
7. Work step by step. Do not try to integrate everything at once.
8. Prefer practical WordPress-friendly solutions.
9. Avoid unnecessary frameworks, build systems, and abstractions.
10. Treat this folder as an internal engineering toolkit.

### Codex expectations
- follow existing project-kit structure
- reuse module docs and prompts
- do not add random files
- do not rename folders without a strong reason
- do not overengineer reusable modules

## RU

Эта папка хранит переиспользуемые строительные блоки для Malibu Exchange и будущих WordPress backoffice-проектов.

Она является частью корня темы Malibu, а не отдельной папкой вне темы.

### Правила
1. Не придумывать новую архитектуру, если уже есть проверенный модуль.
2. Предпочитать извлечение и очистку существующих решений, а не переписывание с нуля.
3. Держать каждый модуль маленьким, изолированным и переиспользуемым.
4. Не смешивать несколько разных задач в одном модуле.
5. Держать промпты, документацию и extracted-код согласованными.
6. Всегда использовать одинаковую структуру модуля:
   - source
   - extracted
   - docs
   - archives
7. Работать поэтапно. Не пытаться интегрировать всё сразу.
8. Предпочитать практичные WordPress-friendly решения.
9. Избегать лишних фреймворков, build-систем и абстракций.
10. Рассматривать эту папку как внутренний инженерный toolkit.

### Ожидания от Codex
- следовать существующей структуре project-kit
- переиспользовать документацию и промпты модуля
- не добавлять случайные файлы
- не переименовывать папки без сильной причины
- не переусложнять переиспользуемые модули
